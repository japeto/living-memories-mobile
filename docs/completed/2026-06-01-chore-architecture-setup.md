# chore: Expo SDK 56 Upgrade & Security Audit

## Summary
Upgrade all dependencies in the LivingMemories Mobile app to their latest versions, specifically migrating from Expo SDK 54 to Expo SDK 56. Focus on resolving any recent security vulnerabilities identified by `npm audit`.

## Scope
- Upgrade `expo` to the latest version (SDK 56).
- Upgrade `react` and `react-native` to versions compatible with Expo SDK 56.
- Upgrade UI and utility libraries (`react-native-paper`, `@expo/vector-icons`).
- Run `npm audit fix` to resolve security vulnerabilities in transitive dependencies.
- Ensure the newly implemented Clean Architecture + MVVM + DI structure is fully compatible with the new versions.

| File | Action | Description |
|------|--------|-------------|
| `package.json` | MODIFY | Update dependency versions. |

## Business Logic / Change Description
1. Wait for the ongoing `tsyringe` and `reflect-metadata` installation to finish.
2. Run `npm install expo@latest` to upgrade the Expo CLI and core package.
3. Run `npx expo install --fix` to auto-align `react`, `react-native`, and other Expo-managed packages to the correct versions for SDK 56.
4. Run `npm install react-native-paper@latest @expo/vector-icons@latest` to upgrade the remaining explicit dependencies.
5. Run `npm audit fix --force` (if necessary) to resolve any high/critical security vulnerabilities.
6. Verify the app builds correctly with the new SDK.
